# 맛봉(Matbong) 서비스 기획서

## 1. 서비스 개요

**맛봉**은 경북 의성 지역의 맛집을 지도에서 확인하고, 카테고리/검색/영업 상태 기준으로 빠르게 찾을 수 있는 **지역 맛집 지도 서비스**다.

현재 코드는 **프론트엔드 단독 웹앱** 구조로 되어 있고, 카카오맵 API를 사용해서 지도와 마커를 보여준다.

- 서비스명: 맛봉
- 주제: 의성 맛집 지도
- 핵심 가치: 사용자가 주변 맛집을 지도에서 빠르게 찾고, 관리자는 맛집 마커를 직접 등록/삭제할 수 있음
- 주요 사용자: 의성 방문자, 학생, 지역 주민, 맛집 관리자
- 현재 구현 방식: HTML, CSS, JavaScript, Kakao Maps API, localStorage

---

## 2. 기획 배경

의성 지역의 맛집 정보는 블로그, 지도 앱, SNS 등에 흩어져 있다.

맛봉은 이런 정보를 한 화면에 모아서 사용자가 **지도 기반으로 맛집을 탐색**하게 만드는 서비스다.

예를 들면 네이버지도나 카카오맵에서 음식점을 찾는 것처럼, 사용자는 지도 위 마커를 누르고 맛집 정보를 확인할 수 있다.

다만 맛봉은 일반 지도 앱보다 더 좁은 목적을 가진다.

**의성 맛집만 보기 좋게 모아둔 전용 지도**라는 점이 핵심이다.

---

## 3. 문제 정의

사용자가 겪는 문제는 다음과 같다.

- 의성 맛집 정보를 한눈에 보기 어렵다.
- 음식 종류별로 맛집을 빠르게 거르기 어렵다.
- 지금 영업 중인지 확인하기 번거롭다.
- 별점이 높은 맛집을 우선적으로 보고 싶다.
- 관리자가 직접 맛집을 추가하고 삭제할 수 있는 간단한 도구가 필요하다.

## 4. 목표

### 사용자 목표

- 맛집을 지도에서 직관적으로 찾는다.
- 검색어로 상호명이나 메뉴 설명을 찾는다.
- 한식, 면류, 고기류, 분식, 카페/디저트 등 카테고리별로 본다.
- 별점 높은 순 또는 최신 등록 순으로 맛집을 본다.
- 현재 영업 중인 맛집만 본다.
- 카카오맵 길찾기로 이동한다.

### 관리자 목표

- 관리자 비밀번호로 로그인한다.
- 지도 클릭으로 맛집 위치를 지정한다.
- 상호명, 카테고리, 별점, 영업시간, 휴무일, 태그, 설명을 등록한다.
- 잘못 등록된 마커를 삭제한다.

## 6. 주요 기능

## 6.1 맛집 지도

카카오맵을 화면 중앙에 표시한다.

지도에는 맛집 데이터가 마커로 표시된다.

마커를 클릭하면 인포윈도우가 열린다.

인포윈도우에는 다음 정보가 나온다.

- 상호명
- 카테고리
- Best 배지
- 별점
- 설명
- 영업시간
- 태그
- 길찾기 버튼
- 관리자 로그인 상태일 때 삭제 버튼

## 6.2 맛집 리스트

현재 필터 조건에 맞는 맛집을 카드 형태로 보여준다.

카드를 클릭하면 해당 맛집 위치로 지도가 이동하고, 해당 마커의 인포윈도우가 열린다.

리스트 카드에는 다음 정보가 나온다.

- 상호명
- Best 배지
- 카테고리
- 별점
- 영업시간

## 6.3 검색

상단 검색창에서 맛집 이름 또는 설명을 검색할 수 있다.

현재 코드 기준 검색 대상은 다음이다.

- `title`: 상호명
- `description`: 설명/메뉴

예시:

- "빽다방" 검색
- "짬뽕" 검색
- "카페" 검색

## 6.4 카테고리 필터

사용자는 카테고리 버튼으로 맛집을 거를 수 있다.

현재 카테고리는 다음이다.

- 전체 보기
- 한식
- 면류
- 고기류
- 분식
- 카페/디저트
- Best 맛집

## 6.5 지금 영업 중 필터

사용자는 **지금 영업 중만 보기** 버튼으로 현재 시간 기준 영업 중인 맛집만 볼 수 있다.

현재 로직은 다음 값을 기준으로 판단한다.

- `startTime`
- `endTime`
- `closedDays`
- `isOpen`

## 6.6 정렬

맛집 리스트는 다음 기준으로 정렬할 수 있다.

- 별점 높은 순
- 최신 등록 순

별점 높은 순에서는 별점이 `0`인 맛집을 맨 뒤로 보낸다.

## 6.7 필터 초기화

검색창 오른쪽의 초기화 버튼을 누르면 다음 값이 초기화된다.

- 검색어
- 카테고리
- 영업 중 필터

## 6.8 관리자 로그인

관리자 버튼을 누르면 비밀번호 입력 모달이 열린다.

현재 비밀번호는 코드에 직접 들어가 있다.

```js
matbong2024
```

로그인 성공 시 `localStorage`에 관리자 상태가 저장된다.

## 6.9 마커 추가

관리자로 로그인하면 마커 추가 버튼을 사용할 수 있다.

마커 추가 흐름은 다음과 같다.

1. 관리자 로그인
2. 마커 추가 모드 On
3. 지도 클릭
4. 맛집 정보 입력 모달 열림
5. 필수값 입력
6. 추가 버튼 클릭
7. 지도와 리스트에 맛집 반영

필수 입력값은 다음이다.

- 상호명
- 카테고리
- 별점

선택 입력값은 다음이다.

- 영업시간
- 휴무일
- 태그
- 설명/메뉴

## 6.10 태그 선택

관리자는 맛집 등록 시 태그를 최대 3개까지 선택할 수 있다.

현재 태그는 다음이다.

- 청결해요
- 맛이 신선해요
- 양이 많아요
- 가성비 좋아요
- 분위기 좋아요
- 친절해요
- 가격이 괜찮아요
- 재방문 의사 있어요
- 가볍게 오기 좋아요
- 거리가 가까워요

## 6.11 마커 삭제

관리자로 로그인한 상태에서 마커 인포윈도우를 열면 삭제 버튼이 보인다.

삭제 버튼을 누르면 확인 모달이 나오고, 확인 시 해당 맛집이 삭제된다.

## 6.12 내 위치 이동

사용자는 내 위치 버튼을 눌러 현재 위치로 지도를 이동할 수 있다.

브라우저의 `navigator.geolocation` 기능을 사용한다.

## 6.13 지도 타입 변경

사용자는 지도 타입을 바꿀 수 있다.

- 지도
- 스카이뷰

## 6.14 지도 확대/축소

사용자는 커스텀 확대/축소 버튼으로 지도 레벨을 변경할 수 있다.

## 12. 현재 코드 기준 이슈

## 12.1 보안 이슈

관리자 비밀번호가 프론트엔드 코드에 그대로 노출되어 있다.

프론트엔드는 사용자가 개발자도구로 코드를 볼 수 있기 때문에, 실제 서비스에서는 이 방식이 위험하다.

실무에서는 관리자 인증을 서버에서 처리해야 한다.

## 12.2 데이터 저장 이슈

현재 맛집 추가/삭제 데이터는 `localStorage`에 저장된다.

`localStorage`는 사용자 브라우저 안에만 저장되는 작은 저장소다.

즉, 내 컴퓨터에서 추가한 맛집은 다른 사람 컴퓨터에는 보이지 않는다.

실제 서비스에서는 데이터베이스가 필요하다.

예시:

- Firebase
- Supabase
- MySQL
- PostgreSQL
- MongoDB

## 12.3 저장 데이터 로드 미연결

`map.js`에 `loadStoredMarkers()` 함수는 있지만, 현재 초기 데이터 로딩 흐름에서는 저장된 마커를 실제로 불러와 `store`에 넣는 연결이 약하다.

그래서 새로고침 후 추가 마커 유지 기능은 의도와 다르게 동작할 가능성이 있다.

## 12.4 API 키 관리 이슈

카카오맵 API 키가 `index.html`에 직접 들어가 있다.

프론트엔드 지도 API 키는 어느 정도 노출될 수 있지만, 실무에서는 도메인 제한과 환경변수 관리가 필요하다.

## 12.5 XSS 가능성

맛집명, 설명, 태그 등이 `innerHTML`로 렌더링된다.

사용자가 악성 HTML을 입력하면 화면에 삽입될 위험이 있다.

실무에서는 사용자 입력값을 화면에 넣기 전에 escape 처리가 필요하다.

## 14.2 백엔드

- 맛집 CRUD API

CRUD는 데이터를 다루는 기본 동작이다.

- Create: 만들기
- Read: 읽기
- Update: 수정하기
- Delete: 삭제하기

맛집 서비스로 비유하면 다음과 같다.

- Create: 맛집 등록

## 15. 성공 지표

서비스가 잘 만들어졌는지 판단하는 기준은 다음이다.

- 사용자가 원하는 맛집을 10초 안에 찾을 수 있는가
- 모바일에서 지도와 리스트가 편하게 보이는가
- 관리자가 실수 없이 맛집을 등록할 수 있는가
- 검색/필터/정렬이 즉시 반응하는가
- 새로고침 후에도 데이터가 유지되는가
- 잘못된 입력으로 화면이 깨지지 않는가
